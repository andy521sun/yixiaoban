# 医小伴 → 在线医疗咨询平台 转型开发计划

## 一、项目概况

**现有基础：**
- 后端 Node.js/Express 7664 行代码
- Flutter 三端（管理后台/患者/陪诊师）共 2.1 万行
- 数据库 20 张表
- 已完成 HTTPS 部署至 https://andysun521.online

**转型目标：**
陪诊师端 → 医生端（在线问诊/接诊/开处方/会诊）
患者端 → 新增在线问诊（图文/视频/报告上传）
管理后台 → 审核/匹配/统计/纠纷处理

**预估总工期：6-8 周（42-56 天）**

---

## 二、详细开发计划（分6个迭代）

### 迭代 1：数据库改造 + 基础后端 API（第 1-7 天）

## 三、防止停工与监工方案

### 3.1 每日站会（自动化）
每天上午9点自动发送进度报告到企业微信：

```yaml
定时任务：
  每天早上9:00 → 自动检查 git 昨天的提交
  每天早上9:05 → 汇总进度发到群里
```

**已经配置了这个定时任务，每天早上你能看到：**
- 昨天完成了什么
- 今天计划做什么
- 是否有阻塞

### 3.2 Git提交规范（可追溯）

```
feat: 新增医生入驻API
fix: 修复问诊单状态不更新
db: 新增consultations表
docs: 更新API文档
```

每次提交自动触发：代码量统计 → 可以看见每天/每周进展

### 3.3 外部可验证的里程碑

| 里程碑 | 验证方式 | 截止时间 |
|---|---|---|
| M1: 数据库改造完成 | `https://andysun521.online/api` 能看到新路由 | D7 |
| M2: 患者端问诊可用 | 浏览器打开 `/patient/` 能完成问诊下单 | D16 |
| M3: 医生端接诊可用 | 浏览器打开 `/companion/` 能接诊回复 | D27 |

| M6: 正式上线 | 全功能生产环境 | D48 |
