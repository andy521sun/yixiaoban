#### 任务6：数据库迁移
**文件：** `/backend/migrations/2024040701_create_orders_table.js`

**表结构：**
```sql
CREATE TABLE orders (
  id VARCHAR(36) PRIMARY KEY,
  user_id VARCHAR(36) NOT NULL,
  hospital_id VARCHAR(36),
  companion_id VARCHAR(36),
  service_type VARCHAR(50) NOT NULL,
  appointment_time DATETIME NOT NULL,
  duration INT NOT NULL COMMENT '服务时长（分钟）',
  price DECIMAL(10,2) NOT NULL,
  status ENUM('pending', 'paid', 'confirmed', 'in_progress', 'completed', 'cancelled') DEFAULT 'pending',
  payment_method VARCHAR(20),
  payment_status ENUM('unpaid', 'paid', 'refunded') DEFAULT 'unpaid',
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  INDEX idx_user_id (user_id),
  INDEX idx_status (status),
  INDEX idx_created_at (created_at)
);
```

## ⚠️ 注意事项

### 技术注意事项
1. **状态管理**：使用Provider管理全局状态，避免setState滥用
2. **网络请求**：使用Dio进行网络请求，做好错误处理和重试
3. **数据持久化**：重要数据使用SharedPreferences或SQLite本地存储
4. **性能优化**：图片使用缓存，列表使用分页加载

### 业务注意事项
1. **支付安全**：支付金额必须与订单金额一致
