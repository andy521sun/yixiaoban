## 二、转型目标

陪诊师端 → **医生端**
患者端新增 → **在线问诊**（文字/图片/视频）
管理后台新增 → **医生审核/问诊统计/处方管理**

## 五、开发原则

1. **不破坏现有功能** — 所有新功能应做到与现有陪诊功能兼容，陪诊师端逐步过渡到医生端
2. **每步可验证** — 每个 API 写完立即 curl 测试，每个页面写完立即构建验证
3. **先 API 后前端** — API 稳定再对接 Flutter，避免前后端来回改
4. **每天至少1次代码提交** — 不管多小的改动，保证进度可追踪
