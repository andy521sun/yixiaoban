## 二、转型目标

- **陪诊师端**（保留原样不动，用户可继续使用）

## 五、开发原则

1. **陪诊端不动医生端新建** — 两者独立，互不影响
2. **不破坏现有功能** — 每步验证现有流程是否仍然正常
3. **每步可验证** — 每个 API 写完立即 curl 测试，每个页面写完立即构建验证
4. **先 API 后前端** — API 稳定再对接 Flutter
5. **每天至少1次代码提交** — 保证进度可追踪
